# Round 5：综合最优 + 多 query 泛化

## 配置

最优组合 = Round 4 胜出：**user + verbose rule + T=0**，example 数 5（Round 2/3 优化）。

测试 5 个不同 query（24-30 chars）验证泛化能力。每个 query 单独 POST 抓自己的 baseline 作对照。

## 结论

1. **length=2 规则一旦遵守，100% span 都 length=2**（19/19）。这是最稳的属性。
2. **±2 字符命中率 94.7%**（18/19）：模型在"压缩"baseline 起点——往往把 6-7 个 baseline 起点合并成 5-7 个，但位置偏差都在 ±2 内。
3. **60% 成功率**：2/5 卡 thinking，说明 user-verbose-T0 仍非确定性。卡死的 2 个 query（"深秋的街道..."、"海边的礁石..."）字数都是 24-25 chars（短），可能是短 query 让模型在 thinking 阶段反复推敲"该不该在第 1 个字就高亮"。
4. **失败 case 仍可控**：thinking 12231/11909 chars，但 max_tokens 4096 兜底截断（不会无限循环），延迟上限约 17 秒。
5. **泛化能力可接受**：3 个成功 query 跨不同主题（深秋/清晨/写作/海边/夜读），质量稳定。失败 query 主题也是随机的（不是"特定句式才卡"）。

## 与 first-round 结论对比

| 维度 | first-round (M3) | 本实验 (M2.7) |
|------|------------------|---------------|
| 纯 few-shot | 卡 thinking | 卡 thinking（复现） |
| + length=2 规则 | 学会（5 spans） | 学会（4-7 spans） |
| 起点 vs baseline | "不完全对齐" | **±2 容差 95-100%** |
| 延迟 | 未量化 | 3.3-17s（平均 10s） |

差距缩小：第一轮发现 length=2 规则是必要条件，本实验进一步精确到 **"user+verbose+T0" 三件套** 是最优组合，且量化了起点重合度。
